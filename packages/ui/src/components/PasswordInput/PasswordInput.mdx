# PasswordInput

Password row used by the four auth flows. Thin wrap over
[`<Input>`](?path=/docs/web-primitives-input--docs) — bakes in
`type="password"`, surfaces a cleaner prop API (`error` collapses to
`isInvalid` + `hint`), and forces a meaningful `autoComplete` so
password managers behave correctly.

## When to use

- The Login screen (`autoComplete="current-password"`).
- The Sign-up screen (`autoComplete="new-password"`).
- The Forgot-password reset step (`autoComplete="new-password"`).
- Any other form that asks the user to enter a password.

## When NOT to use

- A pin / OTP input — use [VerificationCodeInput](?path=/docs/web-primitives-verificationcodeinput--docs).
- A username, email, or generic text input — use [Input](?path=/docs/web-primitives-input--docs).
- An access-token / API-key field — use [Input](?path=/docs/web-primitives-input--docs)
  with `type="text"`; password autofill on a token field is wrong.

## Anatomy

<Canvas of={PasswordInputStories.WithError} />

```tsx
<PasswordInput
  label="Password"
  autoComplete="current-password"
  value={pw}
  onChange={setPw}
  error={errors.password}
/>
```

The visibility toggle (Eye / EyeOff) is rendered by the underlying
kit `<Input>` whenever `type="password"`. It's a real `<button>` so
keyboard users can flip it with Space / Enter.

## Accessibility

- Always pass a `label` — the kit `<Input>` binds it to the field
  via `<label htmlFor>`. Visually-hidden labels are not supported in
  this primitive.
- The visibility toggle is keyboard-operable (`Space` / `Enter`)
  and carries an `aria-label="Toggle password visibility"` from the
  kit. The button does NOT use `aria-pressed` because the visible
  Eye / EyeOff icon swap communicates state visually; the press
  fires a real DOM event each time.
- Pair `error` with descriptive copy ("Password must be at least 8
  characters.") rather than terse error codes — the message is
  surfaced both visually and via `aria-describedby`.
- For Sign up / password-reset, set `autoComplete="new-password"`
  so password managers offer to save the new value (vs. autofilling
  the old one).
